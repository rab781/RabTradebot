## 2024-05-25 - [Optimize Hot-Path Array Methods and Math Spreads]
**Learning:** Using chained array methods (`.map`, `.filter`, `.reduce`) and spread operators (`Math.max(...array)`) on potentially large arrays (like trade histories) causes significant performance overhead from intermediate array creation, garbage collection, and creates a risk of "Maximum call stack size exceeded" errors when spreading massive arrays.
**Action:** Replace these operations with a single, pre-allocated O(N) `for` loop that calculates all necessary metrics (counts, sums, mins, maxes) in a single pass without spreading.
